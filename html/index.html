End-to-end architecture

Where this repo sits in the full platform

Data pipelinePostgreSQL changes are captured by DMS, stored in Bronze, cleaned by Glue, modeled by dbt, and exposed as Gold marts.
Serving layerThe analytics agent queries Gold data through Athena and returns governed answers with SQL, cost, and audit evidence.
FrontendsStreamlit, custom HTML, and Slack/MCP are separate user entry points that share the same backend contract.

Beginner-friendly concept map

Key ideas to understand

Remote state

tfstate→S3→versioning

Terraform needs a durable receipt of what AWS resources it owns.

State locking

DynamoDB→lock→one apply

Locks prevent two applies from editing the same state at once.

OIDC trust

GitHub→short-lived AWS creds→role

CI/CD assumes AWS roles without storing long-lived access keys.
